四、ref全家桶

4.1 ref

  • ref 接收一个内部值，并且返回一个响应式且可变的对象；
  • ref对象有且仅有一个 .value property,指向的是它内部的值。
<template>
  <div>
    <button @click="changeMsg">change</button>
    介绍：{{msg}}
  </div>
</template>

<script setup lang="ts">
import {Ref, ref} from 'vue'
  let msg: Ref<string> = ref('我是北极星')
  const changeMsg = () => {
    msg.value = '我是大白'
    console.log(msg);
  }
</script>
  • 页面更新为 “我是大白”;控制台输出 RefImpl 对象

4.2 为什么ref返回一个对象？

  • ref是一个重载函数，函数内部返回一个 createRef函数
  • createRef函数判断传入的值是否为ref对象，不是的话会返回 RefImpl实例
  • RefImpl内部通过get 和 set 方法对传入的值进行设置和返回

4.3 shallowRef， triggerRef，customRef

4.3.1 shallowRef 创建一个跟踪自身 .value 变化的ref,但是值不会是响应式

  • 使用shallowRef包装的对象，只有到 .value这一级具有响应式；
  • 所以以下修改方式，不生效；控制台输出的是变化的值，但是页面不会更新；
  • 想要具备响应式，只能修改整个对象，写法 obj.value = {name: 'woshidaibai' }
    使用场景： 对于不需要响应式的属性，用shallowRef包裹
<template>
  <div>
    <button @click="changeMsg">change</button>
    {{obj}}
  </div>
</template>

<script setup lang="ts">
import {shallowRef} from 'vue'
  let obj = shallowRef({
    name: '我是大白'
  })
  const changeMsg = () => {
    obj.value.name = 'woshidaibai'
    // obj.value = {name: 'woshidaibai' }
    console.log(obj);
  }
</script>

4.3.2 triggerRef 强制更新页面的dom

  • 使用triggerRef强制更新dom,在修改值后将修改对象传入 triggerRef，页面能够获取修改后得到值。
    <script setup lang="ts">
import {shallowRef, triggerRef} from 'vue'
let obj = shallowRef({
  name: '我是大白'
})
const changeMsg = () => {
  obj.value.name = 'woshidaibai'
  triggerRef(obj)
  // console.log(obj);
}
</script>

    4.3.3 customRef 自定义ref

  • customRef 是一个工厂函数，函数内部要求返回一个对象，对象内部要有get和set方法
  • 点击按钮触发set方法，模板的msg值修改为 “大白”
<template>
  <div>
    <button @click="changeMsg">change</button>
    {{msg}}
  </div>
</template>

<script setup lang="ts">
import {shallowRef, triggerRef,customRef} from 'vue'
  function myRef<T>(value: T) {
    return customRef((track, trigger) => {
      return {
        get() {
          track() //用于收集依赖
          return value
        },
        set(newValue: T) {
          value = newValue
          trigger()
        }
      }
    })
  }
  let msg = myRef<string>('北极星')
  const changeMsg = () => {
    msg.value = '大白'
  }
</script>

4.4 shallowRef造成视图更新的问题

  • 上面的案例中提到了，使用shallowRef包裹的属性，会失去响应式；
  • 但是，如果在一个方法中同时改变响应式数据（就是ref包裹的属性）和shallowRef内的属性，shallowRef中的属性也会变为响应式，从而造成视图的更新。

五、reactive全家桶

  • reactive内部接收复杂数据类型，将其转化为响应式，如果传入基本数据类型，那么会控制台会报错并且将数据直接返回。
  • 传入的数据经过转化返回一个Proxy对象，对象内部则是属性名称，所以页面使用时 obj.property;不需要obj.value.property

5.1数组赋值时注意：

  • 当我们在给响应式的数组赋值时，不能将接口返回数据直接覆盖原来的数组，这样会使原数组丢失响应式;
  • 应该是将返回的数组中的元素添加到响应式数组当中；

方式一：

<script setup lang="ts">
import {shallowRef, triggerRef,customRef, ref, reactive} from 'vue'
const msg = reactive({
  name: '北极星'
})
let arr = reactive<number[]>([])
setTimeout(() => {
  let temp = [1,2,3,4]
  arr = temp //不合理
  arr.push(...temp) //合理
  console.log(arr); //[1,2,3,4]
}, 0)
</script>

方式二：

<script setup lang="ts">
import {shallowRef, triggerRef,customRef, ref, reactive} from 'vue'
type List = {
  list: number[]
}
let msg = reactive<List>({
  list: []
})
setTimeout(() => {
  msg.list = [1,2,33]
  console.log(msg);
}, 0)

</script>

5.2readonly方法

  • 将数据拷贝一份，并且此数据不允许被更改
<script setup lang="ts">
import {readonly, reactive} from 'vue'

let msg = reactive({
  count: 1
})
let copy = readonly(msg)
copy.count++ //报错

</script>

5.3shallowReactive

  • 只有第一层对象的属性为响应式，改变深层次的属性值时，会修改数据但是不会在页面上更新数据

5.3.1使用的注意点：

  • 在dom挂在之前，比如我们在自调用函数内部改变属性值时，shallowReactive不会生效；
  • 在一个方法中同时修改浅层次和深层次属性时，shallowReactive也不生效
<script setup lang="ts">
import {readonly, reactive, shallowReactive} from 'vue'
const test = shallowReactive({
  demo: 'kkk',
  obj: {
    child: {
      name: 'dadadada'
    }
  }
})
//warning： 一个方法同时修改浅深层次属性时，两者属性都会响应式更新
const changeMsg = () => {
  test.demo = 'kkk被改了。。。'
  test.obj.child.name = '这回里面被改了' //需要单独写，并且正常来说也应该拆分出来，一个函数做一件事
}
const changeInner = ()=> {
  test.obj.child.name = '这回里面被改了'
}
/*
  1. 函数自调用，这是dom还没挂载结束，shallowReactive不会生效；
  2. 应该在页面手动触发变更事件
*/
changeMsg()
changeInner()
</script>

六、to全家桶（toRef,toRefs,toRaw）

6.1 toRef

  • 它接收一个属性，将其变为响应式属性
<script setup lang="ts">
import {toRef, toRefs, toRaw} from 'vue'
const obj = {
  name: '北极星',
  age: 15
}
const state = toRef(obj, 'age') //age属性变为响应式
</script>

6.2 toRefs

  • 我们在将对象解构的时候，依然希望响应式里面对象的属性解构出来的每个属性都是响应式的；
<script setup lang="ts">
import {toRef, toRefs, toRaw, reactive} from 'vue'

const obj = reactive({
  name: '北极星',
  age: 15
})
// const {name, age} = obj
  // 解构出来的属性不是响应式
// 这时候需要toRefs
const {name, age} = toRefs(obj)
console.log(name, age); 
const changeMsg = () => {
  name.value = '大白'
  age.value++
}
</script>

6.3 toRaw

  • 将一个响应式对象改为非响应式
<script setup lang="ts">
import {toRef, toRefs, toRaw} from 'vue'
const obj = reactive({
  name: '北极星',
  age: 15
})
const res = toRaw(obj)
console.log(res); // 输出 {name:'北极星', age: 15}
</script>

八、Wath监听函数

8.1基本用法

<script setup lang="ts">
import {watch, ref} from 'vue'
const msg = ref<string>('北极')

watch(msg, (newVal, oldVal) => {
  console.log('新值：', newVal, '旧的值：',oldVal);
})
</script>

8.2监听多个属性时

// 第一个参数传入一个数组，数组中元素为监听的属性
watch([msg1,msg2], (newVal, oldVal) => {
  console.log('新值：', newVal, '旧的值：',oldVal);
})

8.3watch函数配置

8.3.1 监听ref响应式对象

  • 当监听的属性层级超过一层时，需要配置第三个参数deep(深度监听)，immediate(监听立即执行);
  • 这里有个bug,就是deep开启后，新值和旧值输出都一致（原因？TODO）
<template>
  <div>
      <input type="text" v-model="msg.far.son.name" />
  </div>
</template>
<script setup lang="ts">
import {watch, ref} from 'vue'
const msg = ref({
  far: {
    son: {
      name: '北极星'
    }
  }
})
watch(msg, (newVal, oldVal) => {
  console.log(msg);
  console.log('新值：', newVal, '旧的值：',oldVal);
}, {
  deep: true
})
</script>

8.3.2 监听reactive响应式对象

  • 监听reactive响应式对象时，默认就开启了深度监听deep，这时使用watch函数不需要配置deep;
<script setup lang="ts">
import {watch, ref, reactive} from 'vue'
const msg = reactive({
  far: {
    son: {
      name: '北极星'
    }
  }
})

watch(msg, (newVal, oldVal) => {
  console.log(msg);
  console.log('新值：', newVal, '旧的值：',oldVal);
}, {
  immediate: true
})
</script>

8.3.3 监听对象中指定属性

  • 对watch函数，第一个参数指定返回被监听对象的指定属性；这时候除了对象中指定属性改变，其他属性改变则不会触发监听函数；
<script setup lang="ts">
import {watch, ref, reactive} from 'vue'
const msg = reactive({
  far: '大白',
  son: '小黑'
})
watch(() => msg.son, (newVal, oldVal) => {
  // 只有son属性变化，才会触发监听函数内的逻辑
  console.log('新值：', newVal, '旧的值：',oldVal);
}, {
  immediate: true
})

</script>

九、watchEffect高级侦听器

9.1 首次加载就会执行侦听器里的逻辑代码

<script setup lang="ts">
import {watchEffect, ref, reactive} from 'vue'
const msg1 = ref('这是大白')
const msg2 = ref('这是我的')

watchEffect(() => {
  console.log(msg1.value);
  console.log(msg2.value);
})
</script>

9.2 watchEffect内部自定义回调函数

  • watchEffect内部可以接收一个回调函数，用于在数据变化之前做预处理；
  • 这个预处理函数除了首次加载，每次数据变化时会优先执行。
<script setup lang="ts">
import {watchEffect, ref, reactive} from 'vue'
const msg1 = ref('这是大白')
const msg2 = ref('这是我的')
// 执行顺序
watchEffect((onInput) => {
  console.log(msg1.value); // 2
  console.log(msg2.value); // 3
  onInput(() => {
    console.log('do something...'); // 1 
  }) 
})
</script>

9.3 定义停止函数

<template>
  <div>
    <div>
      <input type="text" v-model="msg1" />
      <input type="text" v-model="msg2" />
    </div>
    <button @click="stopWatch">停止监听</button>
  </div>
</template>
<script setup lang="ts">
import {watchEffect, ref, reactive} from 'vue'
const msg1 = ref('这是大白')
const msg2 = ref('这是我的')

const stop = watchEffect((onInput) => {
  console.log(msg1.value);
  console.log(msg2.value);
  onInput(() => {
    console.log('do something...');
  }) 
})

const stopWatch = () => stop()
</script>

9.4 watchEffect的配置项

<script setup lang="ts">
import {watchEffect, ref, reactive} from 'vue'
const msg = reactive({
  far: '大白',
  son: '小黑'
})

const msg1 = ref('这是大白')
const msg2 = ref('这是我的')

const stop = watchEffect((onInput) => {
  const item: HTMLInputElement = document.querySelector('#msgId') as HTMLInputElement

9.5 调试函数 onTrigger,在监听属性改变后会触发监听函数；用于开发时断点调试

import {watchEffect, ref, reactive} from 'vue'

十一、父子组件传值

11.1 父传子，子传父

  • 简单数据类型直接传值，复杂数据类型v-bind传值；
  • 子传父，需要子组件派发一个事件，父组件监听并且自定义处理函数进行接收；
  • 父组件获取子组件实例时使用ref='自定义一个属性名'，并且子组件需要指定暴露 defineExpose 那些数据
<template>
  <div class="layout">
    <Menu ref="menuRef" :list="data" title="这是一套衣服" @on-click="handleClick"></Menu>
    <div>---{{arr}}</div>
    <div class="layout-right">
      <Header></Header>
      <Content></Content>
    </div>
  </div>
</template>
<script setup lang="ts">
import { ref } from 'vue';
import Content from '../layout/content/index.vue'
import Header from '../layout/header/index.vue'
import Menu from '../layout/menu/index.vue'

const data = ref([1,2,3])
const arr = [] as number[]

// 获取 menu组件的实例，子组件可以选择性暴露数据；父组件只能拿到子组件暴露出的数据
const menuRef = ref(null)

const handleClick = (val: number[]) => {
  console.log(val);
}
</script>

//子组件
<template>
  <div class="menu">这是菜单
    <div style="color: red" >{{title}}</div>
    <div>{{list}}</div>
    <div>
      <button @click="toParent">派发</button>
    </div>
  </div>
</template>

<script setup lang="ts">
import { reactive } from 'vue';
  type Props = {
    title: string,
    list: number[]
  }
  defineProps<Props>()
  const temp = reactive([111,222])
  const emit = defineEmits(['on-click'])
  const toParent = () => {
    emit('on-click', temp)
  }  
  // 父组件获取当前组件实例时能拿到的数据，暴露给父组件的数据
  defineExpose({
    temp
  })  

</script>

11.2 当不确定父组件是否传值时，子组件可设置默认值

  • withDefaults 以及 defineProps
<template>
  <div class="menu">这是菜单
    <div>---{{title}}</div>
    <div>---{{data}}</div>
  </div>
</template>
<script setup lang="ts">
import { reactive } from 'vue';
  type Props = {
    title?: string,
    data?: number[]
  }
  withDefaults(defineProps<Props>(), {
    title: '默认标题',
    data: () => [11, 22, 33]
  })
</script>

十二、全局组件，递归组件

12.1 注册全局组件

  • 定义的全局组件需要在 main.ts文件中引入，注册时需要在app挂载之前，链式调用的写法；
  • 全局组件的组件在其他页面使用时，不需要导入，直接以标签的方式使用即可。
import Card from './components/card/index.vue'
// 注册组件时需要在挂载之前
createApp(App).component('Card', Card).mount('#app')

12.2 递归组件

  • 递归组件适用于渲染嵌套的数据结构；
  • 渲染逻辑需要终止条件，否则会造成内存泄漏；
  • 引用递归组件（本质就是引用自身）时有两种方式，1.使用import方式导入 2.定义一个script标签，导出组件名；
  • 绑定点击事件时，需要注意冒泡事件。
<template>
  <div>
    <div
      v-for="(item, index) in data"
      :key="index"
      style="margin-left: 10px; color: red;"
      @click.stop="clickItem(item)"
    >
      {{ item.name }}
      <TreeItem
        @on-click="clickItem"
        :data="item.children"
        v-if="item.children?.length"
      ></TreeItem>
    </div>
  </div>
</template>

<script setup lang="ts">
// 引入递归组件，实际就是引用自身；这是写法一
// import TreeItem from './index.vue'

type TreeList = {
  name?: string | undefined
  icon?: string | undefined
  children?: TreeList[] | []
}

type Props = {
  data?: TreeList[]
}
defineProps<Props>()

const emit = defineEmits(['on-click'])
const clickItem = (item: TreeList) => {
  console.log(item);
  // alert(item)
  emit('on-click', item)
}

</script>

// 写法二
<script lang="ts">
export default {
  name: 'TreeItem'
}

20.1 通过父组件除了其中一个子组件派发出的事件

  • 父组件通过 v-on 监听到事件，并且A 组件传递出的参数通过 v-bind 的形式传入 B 组件；
  • B组件通过 props 接收父组件传入的值，由此通过父组件作为桥梁实现兄弟组件传值；

    20.2 事件总线（Bus）

  • 主要原理是利用js的发布订阅模式实现;
  • 定义list全局对象用于缓存回调函数， emit 用于触发事件， on 方法用于事件监听；
  • 当on方法执行时，先从缓存list中查找传入的事件名是否已经存在对应的缓存数组，1.不存在则创建向list中存入一个 事件名: 空数组 的key-value结构数据，并且将传入的 回调函数 callback 存入数组中；2.存在的话，那么根据事件名取出数组，将回调函数存入即可；
  • 当emit方法执行时，根据传入的事件名取出回到函数数组，遍历数组中的回调函数逐个执行；

    20.3 栗子

  • 默认导出一个Bus实例

    20.3.1 事件总线实例

type BusClass = {
  emit: (name: string) => void
  on: (name: string, callback: Function) => void
  off: (name: string) => void
}

type PramsKey = number | string | symbol
type List = {
  [key: PramsKey]: Array<Function>
}

class Bus implements BusClass {
  list: List
  constructor() {
    this.list = {}
  }
  emit(name: string, ...args: Array<any>) {
    let fnList: Array<Function> = this.list[name]
    fnList.forEach(fn => {
      fn.apply(this, args)
    })
  }
  on(name: string, callback: Function) {
    let fn: Array<Function> = this.list[name] || []
    fn.push(callback)
    this.list[name] = fn
  }
  off(name: string) {
    const res = this.list[name] || []
    if (res.length) Reflect.deleteProperty(this.list, name)
  }
}

export default new Bus()

const vMove: Directive<any, void> = (el: HTMLElement, dir: DirectiveBinding) => {
  // 获取操作的dom元素 也就是 box
  let moveElement: HTMLDivElement = el.firstElementChild as HTMLDivElement

    //  获取鼠标按下时在div中x，y轴的距离

      }

        window.innerHeight - parseInt(el.style.height)
      ) {
        el.style.top =
          window.innerHeight - parseInt(el.style.height) + 'px'
      }
      // console.log('mouse...', e);

    })

27.1 scoped

  • vue中的scoped通过在dom结构以及css样式上添加唯一不重复的标记:data-v-hash的方式，保证样式的唯一（实现是由postcss转译实现），达到样式私有化；
  • scoped渲染规则：
  1. 给html的dom节点添加一个不重复的data属性（形如：data-v-123）来标识其唯一性
  2. 在每句css选择器的末尾（编译后的css语句）加上一个当前组件的data属性选择器（如[data-v-123]）来私有化样式
  3. 如果组件内部包含有其他组件，只会给其他组件的最外层标签上加上当前组件的data属性
  • Postcss会给一个组件中所有dom添加一个唯一的动态属性data-v-xxx,然后，给css选择器额外添加一个对应的属性选择器来选择该组件中的dom，这种做法使得样式只作用于含有该属性的dom-组件内部dom，从而达到样式模块化的效果；

二十八、Pinia

28.1 与vuex的差异，特点

  • 完整的ts 的支持;
  • 足够轻量，压缩后的体积只有1kb左右;
  • 去除 mutations，只有state,getters,actions;. actions支持同步和异步;
  • 代码扁平化没有模块嵌套，只有store的概念，store之间可以自由使用，每一个store都是独立的·无需手动添加store，store一旦创建便会自动添加;
  • 支持Vue3和Vue2

28.5.2 $subscripe 与$onAction

  • $subscripe用于监听state中值的变化;类似于 watchEffect
  • $onAction监听actions方法触发
<script setup lang="ts">
  import {useTestStore} from './store'
  import {storeToRefs} from 'pinia'
  
  const test = useTestStore()
  // state值改变时触发,类似于 watchEffect
  test.$subscribe((args, state) => {
    console.log('subscribe..', args, state);
  }, {
    detached: true,
    deep: true,
    flush: 'post'
  })

  // 第二个参数，设置为true时表示组件销毁后该方法依旧存活
  // args参数中还有 onError等方法，用于捕获失败信息等
  test.$onAction((args) => {
    args.after(() => {
      console.log('after。。。');
    })
    console.log('onaction', args);
  }, true)

  // 修改值的方式
  const change = () => {
    test.setUser('战斗机')
  }
  const reset = () => {
    test.$reset()
  }
</script>

28.6 pinia插件

  • vuex与pinia会在页面刷新时丢失状态，因为刷新时组件生命周期等重新执行，数据被重置；但是有时候我们希望刷新后依旧保留，会将数据存储在localStore 或是 sessionStore中
  • 通过自定义持久化插件实现以上功能
import { createApp, toRaw } from 'vue'
import './style.css'
import App from './App.vue'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import {createPinia, PiniaPluginContext} from 'pinia'
type Options = {
  key: string
}
// 导出的是个插件
const store = createPinia()

const __defaultKey__ = 'beijixing'
const setStorage = (key: string, val: any) => {
  localStorage.setItem(key, JSON.stringify(val))
}
const getStorage = (key: string) => {
  return localStorage.getItem(key) ? JSON.parse(localStorage.getItem(key) as string) : {}
}

const piniaPlugin = (option: Options) => {
  // 接收 pinia 返回值
  return (context: PiniaPluginContext) => {
    const { store } = context
    const data = getStorage(`${option.key ?? __defaultKey__}-${store.$id}`)
    // 监听store中属性值的变化,将值存入 localStorage
    store.$subscribe((args) => {
      setStorage(`${option.key ?? __defaultKey__}-${store.$id}`, toRaw(store.$state))
    })
    return {
      ...data
    }
  }

}
// 注册pinia插件
store.use(piniaPlugin({
  key: 'pinia'
}))

createApp(App)
.use(store)
.use(ElementPlus)
.mount('#app')
