使用webpack搭建vuecli

vuecli webpack
vuecli webpack
发布日期:   2022-06-12
文章字数:   5.3k
阅读时长:   26 分

一、基础步骤

  1. npm init 初始化 package.json文件
  2. 创建src目录->在此目录下创建 main.js 文件作为入口文件
  3. 在 package.json 文件内新增打包命令
  4. 执行 npm run build 查看是否打包成功 （成功的话dist文件夹下会新增打包的文件）

    二、webpack基础配置

    1、新建文件

    • 在项目根目录下新增build目录，并且创建webpack.config.js文件

      2、配置文件

  • webpack.config.js 写入
// 引入核心模块path，用户获取文件路径
const path = require('path')
// 导出配置信息
module.exports = {
    // 设置模式
    development， production 开发、生产模式
    默认生产模式，也可以设置为 none */
    mode: 'development',
    // 设置入口文件路径
    entry: path.resolve(__dirname, '../src/main.js'),
        // 设置输出信息
        output: {
            // 输出文件名
            filename: 'budle.js',
                path: path.resolve(__dirname, '../build')
        }
}

3、更改打包

  • 在package.json文件中更改打包命令，并且执行npm run build
    "build": "webpack --config build/webpack.config.js"

4、缓存以及自动引入js

  • HTML自动引入打包后的js文件
  1. 使用 webpack 来打包我们的模块化后的应用程序，webpack 会生成一个可部署的 /dist 目录，然后把打包后的内容放置在此目录中。
  2. 只要 dist 目录中的内容部署到 server 上，client（通常是浏览器）就能够访问此 server 的网站及其资源。而最后一步获取资源是比较耗费时间的，所以浏览器会使用 缓存 。
  3. 浏览器通过命中缓存，以降低网络流量，使网站加载速度更快；但是如果我们在部署新版本时不更改资源的文件名，浏览器可能会认为它没有被更新，就会使用它的缓存版本。由于缓存的存在，当你需要获取新的代码时，就会显得很棘手。
  • 基于以上我们不能每次打包生成同样的文件名，也不会每次打包后去更改 index.html 文件里面引入的打包后的js文件
  • 每次打包生成不同名称的文件 html-webpack-plugin
// 安装
npm i -D html-webpack-plugin@4

4.1、新增 public 文件夹，创建 index.html 作为我们的模板文件

  • 在webpack.config.js中新增配置
  • 修改输出文件名，确保每次生成的js文件名称不重复
  • 新增plugin配置，查看生成的index.html文件中是否自动引入了js文件

5、多入口文件的开发

  • 定义：多入口文件就是设置多个入口文件和输出对应打包后的文件

  1. 配置单个入口文件时： entry为一个属性

  2. 配置多个入口文件： entry需要配置成一个对象

    2.1 对象里面的 属性名 就是该入口文件的标识，属性值就是对应的文件路径，输出时需要在对应的html文件中引入，我们可以通过配置多个html-webpack-plugin的方式实现。

    2.2 在html-webpack-plugin中需要 设置filename来设置输出的html文件名，chunks来关联上面entry里面的入口文件

5.1、配置栗子

Talk is cheap,show the code

// 引入核心模块path，用户获取文件路径
const HtmlWebpackPlugin = require("html-webpack-plugin")
const path = require('path')

// 导出配置信息
module.exports = {
  /* 设置模式
  development， production 开发、生产模式
  默认生产模式，也可以设置为 none */
  mode: 'development',
  // 设置入口文件路径
  entry: {
    main: path.resolve(__dirname, '../src/main.js'),
    other: path.resolve(__dirname, '../src/other.js')
  },
  // 设置输出信息
  output: {
    // 输出文件名
    filename: '[name].[hash:8].js',
    path: path.resolve(__dirname, '../build')
  },
  plugins: [
    new HtmlWebpackPlugin({
      // 设置模板
      template: path.resolve(__dirname, '../public/index.html'),
      filename: 'index.html',
      chunks: ['main']
    }),
    new HtmlWebpackPlugin({
      template: path.resolve(__dirname, '../public/other.html'),
      filename: 'other.html',
      chunks: ['other'] //关联上面entry的类名
    })
  ]

}

5.2、清除残余文件，插件 clean-webpack-plugin

  • 由于每次打包都会生成新的文件，旧的文件依旧残留在dist目录里；我们需要的是新的生成之后删除旧的文件
npm i -D clean-webpack-plugin
//引入
const { CleanWebpackPlugin } = require("clean-webpack-plugin")
// 注册
plugins: [ new CleanWebpackPlugin()  ]

三、处理除了js以外的文件

  • webpack默认只能处理 js 和 json 文件，要想对其他文件生效，需要加载对应的loader

1、引用css文件

  1. 新建assets静态文件夹，新增normal.css文件
  2. 使用预处理器如 less sass 等，需要安装对应加载器以及新增 .less或 .scss文件
//**注意版本号是否兼容**
npm i -D style-loader@2 css-loader@3  
//如果使用预处理器，则需要多安装相应的预处理器和loader
npm i -D less less-loader@5
  • 在webpack.config.js文件中新增 loader配置
 module.exports = {
    module: {
        rules: [
          {
            // 正则匹配所有 .css 结尾的文件
            test: /\.css$/,
            // 匹配到之后使用的loader，从右向左解析
            use: ['style-loader', 'css-loader']
          },
          {
            test: /\.less$/,
            use: ['style-loader', 'css-loader', 'less-loader']
          }
        ]
  }
}

2、添加浏览器前缀

  • **postcss-loader autoprefixer**
//注意版本号
npm i -D postcss-loader@4 autoprefixer@8

1.在module中配置postcss-loader
    module: {
        rules: [   
          {
              //正则匹配所有.css结尾的文件
              test:/\.css$/,
              //匹配到之后使用的loader,从右向左解析
              use:['style-loader','css-loader','postcss-loader']
          },
          {
              test:/\.less$/,
              use:['style-loader','css-loader','postcss-loader','less-loader']
          }
        ]
     }
2. 使autoperfixer生效的方式
  2.1方式一：在1.基础上还需要在项目根目录新增postcss.config.js配置文件；添加如下配置
    module.exports = {
      plugins:[require('autoprefixer')]
    }
  2.2方式二：在1. 的基础上修改配置如下
     use:['style-loader','css-loader',{
         loader: 'postcss-loader',
         options: {
             plugins:[require('autoprefixer')]
         }
     },'less-loader']
  • 在index.html中样式被添加到了style标签中；但如果样式文件过多的时候；index文件就会过于混乱，咱们需要把css文件拆分出去。通过外链的方式引入。

4、打包其他文件：图片，字体，媒体等

1. 对图片、字体、媒体等静态资源一般使用url-loader进行处理后放入打包目录里面，
2. url-loader:如果文件小于限制的大小会转成base64编码,否则使用file-loader将文件移动到输出的目录中
3. url-loader内置了file-loader所以不需要安装file-loader

npm i -D  url-loader@4 file-loader@4

rules:[
       //处理图片
    {
        //正则匹配图片格式
        test:/\.(jpe?g|png|gif)$/i,
        use:[
            {
                loader:'url-loader',
                options:{
                    //限制文件大小
                    limit:10240,
                    //超出上面限制之后使用的loader ext是文件本来的扩展名
                    name:'image/[name].[hash:8].[ext]'
                }
            }
        ]
    },
]

执行npm run build 
可以看到dist文件下下面多了image文件夹，里面包含刚才在main.js里引入的图片，其他静态资源配置类似。

四、搭建一个vue-cli

1、准备

  1. vue是单文件组件，使用的是.vue文件，需要使用vue-loader来解析.vue文件
  2. 每个vue文件里面包含三个部分template(模板)、script(逻辑)、style(样式)；需要使用vue-template-compiler来编译模板,vue-style-loader来应用样式

2、依赖安装

  • npm i -D vue-loader vue-template-compiler vue-style-loader
  • npm i -S vue

2.1、配置 webpack-dev-server 进行热更新

  • 热更新允许在运行时替换，添加，删除各种模块，而无需进行完全刷新重新加载整个页面。热更新中的热我们可以理解为这个模块正在运行中，热替换就是将正在运行的模块进行替换。
npm i -D webpack-dev-server
webpack.config.js新增配置
1. 配置开发服务器
  devServer: {
    port: 8080,
    hot: true, //开启热更新
    contentBase: '../dist' //告诉服务器内容来源
  },
2. src目录下新增App.vue作为根组件
3. 修改public文件下的index.html;它添加 div
4. package.json 新增配置 "dev": "webpack-dev-server --config build/webpack.config.js --open"
5. 更改webpack-cli版本号 npm i -D webpack-cli@3

3、区分开发环境与生产环境

  • 在build目录下新增两个文件,webpack.dev.js webpack.prod.js
  • 开发环境主要是要实现热更新,不需要压缩代码，完整的sourceMap
  • 生产环境需要压缩代码、提取css文件、合理的sourceMap、分割代码等
1. 使用webpack-marge 合并配置采用4版本
2. 使用copy-webpack-plugin 拷贝静态资源采用5版本
3. 使用optimize-css-assets-webpack-plugin  压缩css采用5版本
4. 使用uglifyjs-webpack-plugin 压缩js采用2版本

3.1、开发环境配置

import Vue from 'vue'
