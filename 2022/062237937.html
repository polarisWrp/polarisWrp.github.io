Js闭包

作用域 Js闭包
发布日期:   2022-06-22
文章字数:   1.4k
阅读时长:   5 分

1. 什么是闭包

1.1 定义：

  • 闭包指的是有权访问另一个函数作用域中的变量的函数

    1.2 Js代码执行过程

  • 执行过程分为 编译阶段 和 代码执行 阶段
  1. 代码编译阶段: 由编译器完成，将代码翻译为可执行的代码，并且确定作用域规则
  2. 执行阶段由Js引擎完成：执行可执行的代码，创建 执行上下文

    2. 作用域和作用域链

    2.1 作用域

  • 作用域包括 全局作用域 和 函数作用域
  • Js中将作用域定义为一套规则，用于管理引擎如何在 当前作用域 以及 嵌套子作用域 中根据 标识符名称 进行变量（变量名或者函数名）查找

    2.2 作用域链

  1. 当访问一个变量，编译器执行代码是从当前的作用域查找是否存在这个标识符(指变量名或函数名)，没找到就会去父作用域查找，
    直到全局作用域为止。

  1. 作用域链就是由当前作用域与上层作用域的一系列变量对象组成，保证了当前执行的作用域符合 访问权限的变量 和 函数的有序访问
    函数在调用激活时，会开始创建对应的执行上下文，在执行上下文生成的过程中，变量对象，作用域链，以及this的值会分别被确定。

3. 闭包如何产生

3.1产生的本质

  • 当前环境存在指向对父级作用域的引用

    3.2 闭包

  • 闭包是一个特殊的对象，由 执行上下文和在执行上下文中创建的函数组成；当函数执行时，如果访问了上下文中变量对象的值，闭包就产生了。

3.2.1栗子1

  function foo() {
    var a = 20;
    var b = 30;
    function bar() {
      return a + b;
    }
    return bar;
  }
  var bar = foo();
  bar();
  1. 如上案例：首先有 执行上下文foo，在foo中定义了函数bar，而通过return返回bar的方式使bar执行。
当bar执行时，`访问了foo内部的变量a，b`。因此这个时候`闭包产生`。


2. JavaScript的自动`垃圾回收机制`: 当一个`值在内存中失去引用`时，立即回收机制会根据特殊算法找到并将其回收;
3. 函数的 `执行上下文` 在执行完毕之后，`生命周期结束`，该函数的执行上下文就会`失去引用`;其占用的`内存空间`很快就会被 垃圾回收器 释放,可是 `闭包的存在，会阻止这一过程`。

3.2.2栗子2

var fn = null;
function foo() {
    var a = 2;
    function innnerFoo() {
        console.log(a);
    }
    fn = innnerFoo; // 将 innnerFoo的引用，赋值给全局变量中的fn
}
function bar() {
    fn(); // 此处的保留的innerFoo的引用
}
foo();
bar()
  1. foo()执行完毕,生命周期结束，所占内存本该被垃圾收集器释放。 但是通过 fn= innerFoo 赋值给了全局变量，
    函数innerFoo引用被保留。
  2. 函数 fn 在bar 中执行，innnerFoo函数内访问了外层a变量；foo的变量对象，也被保留了下来。
  3. 所以 可以称 foo 为闭包函数

3.3产生闭包原因

  • 返回函数
  • 函数作为参数传递

4. 创建函数

4.1函数分类

  • 函数分为 具名函数 和 匿名函数

4.2函数创建方式

4.2.1声明函数

function fn() {}

4.2.2创建匿名函数表达式

/**
    * 获取指定函数的函数名称（用于兼容IE）
    * @param {Function} fun 任意函数
    */
function getFunctionName(fun) {
    if (fun.name !== undefined)
        return fun.name;
    var ret = fun.toString();
    ret = ret.substr('function '.length);
    ret = ret.substr(0, ret.indexOf('('));
    return ret;
}

4.2.3创建一个变量，变量的内容为表达式

var fn = function() {}
// 并且由此创建出来的函数没有函数名, 即为匿名函数
getFunctionName(fn).length  // 0

4.2.4创建具名函数

//创建一个变量，内容为一个带有名称的函数
var fn = function getName(){};

// 注意： 
具名函数表达式的 函数名 只能在创建函数内部使用

1. 即采用此种方法创建的函数在函数外层只能使用 fn 不能使用 getName 的函数名。
2. getName 的命名只能在创建的函数内部使用

// 在对象内定义函数如
var o={ fn : function (){…} }，也属于函数表达式

4.2.5Function构造函数

  • 可以给 Function 构造函数传一个函数字符串，返回 包含这个字符串命令的函数，此种方法创建的是匿名函数。
const res = new Function('alert(1)')   
console.log(res);

4.2.6自执行函数

  • 自执行函数属于上述的 函数表达式
(function(){alert(1);})();
(function fn1(){alert(1);})();

5.作用域链

5.1 函数作用域问题

  • 对象内部的函数表达式
    var o={
fn:function (){
  console.log(fn);
}
};
o.fn();//ERROR报错
  • 非对象内部的函数表达式
    var fn=function (){
console.log(fn);
};
fn(); //function (){console.log(fn);};正确

    5.2总结

  • 使用 var或是’非对象内部的函数表达式内’，可以访问到存放当前函数的变量；在对象内部的不能访问到。
  1. 因为函数作用域链的问题，采用var的是在外部创建了一个fn变量，函数在内部寻找不到fn后向上作用域查找fn，
  2. 而在创建对象内部时，因为没有在函数作用域内创建fn，所以无法访问。
