总结使用el-upload上传各种静态文件

element-ui vue2 typescript
发布日期:   2022-08-07
文章字数:   4.7k
阅读时长:   19 分

1、写在前面

  • 在此之前，对于el-upload组件的二次封装大多用于图片的上传，也没有深入了解这个上传控件以及上传文件时一些js原生api,例如FileReader、FormData、Video，以此一文总结文件上传组件功能优化项、文件上传关键点以及使用el-upload上传各种静态资源文件。

2、上传文件需要了解的Js原生api

2.1 FileReader

  • MDN： https://developer.mozilla.org/zh-CN/docs/Web/API/FileReader

  • 需要把文件转为后端需要的入参格式，比如base64或是二进制，这里需要转为base64;

  • readAsDataURL函数接收一个Blob类型参数,在onloadend函数事件在读取操作结束时拿到结果（base64数据）;

  • 转换之后根据后端需求，使用split等方法分割出需要的参数。

const reader = new FileReader()
reader.readAsDataURL(file.raw)
reader.onloadend = () => {
    // 图片的 base64 格式, 可以直接当成 img 的 src 属性值
    const dataURL = reader.result as string
}

2.2 FormData

  • MDN:https://developer.mozilla.org/zh-CN/docs/Web/API/FormData

  • FormData提供了一种表示表单数据的键值对key/value的构造方式；

  • 将数据通过XMLHttpRequest.send() 方法发送出去，如果送出时的编码类型被设为 "multipart/form-data"，它会使用和表单一样的格式；

  • 根据情况，后端需要的file入参一般为Blob类型，那么使用el-upload上传时直接取file.raw就可；

    使用方式形如：

    const param = new FormData()
//设置属性
param.append('file', file.raw)
param.append('name', file.name)
// 读取属性
param.get('属性名')
const url = 'https://xxx.com'
//使用axios的post请求上传文件
axios.post(url, param).then((res: any) => { todo.... })

2.3 Video

  • MDN:https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/video

  • 视频播放预览只需要给video标签的 src属性赋值文件路径参数就可；

  • video标签可以设置自动播放，展示控制面板等等，根据需求设置默认属性；

  • 在不支持video标签的浏览器中，video标签中的内容会降级处理，也就是说类似于 if else-if 的处理方式；

形如：

//当第一个标签内容无法展示，则播放第二个，都无法播放则展示提示语内容
<video controls width="250">
    <source src="/media/cc0-videos/flower.webm"
            type="video/webm">

            type="video/mp4">
    Sorry, your browser doesn't support embedded videos.
</video>
  • 当video标签作为内嵌组件插入到父组件但是我们还想获取它的实例对象时，需要保证外层组件已经渲染完毕，这样才能拿到video实例对象，从而使用play,pause等方法；

形如：

// 这是预览视频的子组件

3、文件下载以及Base64解码方法

3.1 文件下载（pdf, excel）

//1. window.open
window.open(url, '_blank')

//2. 下载文件 支持跨域
function downloadFile(file: any) {
    const xhr = new XMLHttpRequest()
    // 请求方式， 请求地址， 是否为异步
    xhr.open('GET', file.url, true)
    xhr.responseType = 'blob'
    xhr.onload = () => {
        //创建a标签
        const element = document.createElement('a')
        element.style.display = 'none'
        //创建下载链接
        const url = window.URL.createObjectURL(xhr.response)
        element.href = url
        //下载后文件名称
        element.download = file.fileName
        element.click()
        //下载完成移除dom元素
        document.body.removeChild(element)
        //释放内存
        window.URL.revokeObjectURL(url)
      }
    xhr.send()
}

3.2 base64转为二进制

function base64_decode(data) {
  var b64 = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/=";
  var o1,
    o2,
    o3,
    h1,
    h2,
    h3,
    h4,
    bits,
    i = 0,
    ac = 0,
    dec = "",
    tmp_arr = [];
  if (!data) {
    return data;
  }
  data += "";
  do {
    h1 = b64.indexOf(data.charAt(i++));
    h2 = b64.indexOf(data.charAt(i++));
    h3 = b64.indexOf(data.charAt(i++));
    h4 = b64.indexOf(data.charAt(i++));
    bits = (h1 << 18) | (h2 << 12) | (h3 << 6) | h4;
    o1 = (bits >> 16) & 0xff;
    o2 = (bits >> 8) & 0xff;
    o3 = bits & 0xff;
    if (h3 == 64) {
      tmp_arr[ac++] = String.fromCharCode(o1);
    } else if (h4 == 64) {
      tmp_arr[ac++] = String.fromCharCode(o1, o2);
    } else {
      tmp_arr[ac++] = String.fromCharCode(o1, o2, o3);
    }
  } while (i < data.length);
  dec = tmp_arr.join("");
  dec = utf8_decode(dec);
  return dec;
}

4、使用el-upload进行文件上传

  • 由源码可知，onSuccess, onPreview等函数只有在 auto-upload为true时才会执行，默认也为true；
  • 在设置auto-upload为false时，进行上传文件类型校验，逻辑转换等可以在 on-change函数中编辑；
  • 对于限制文件只能上传单个的，可以通过动态绑定样式控制上传框的显隐，自定义类名挂在 el-upload标签上，display:none;
  • 组件内部可以在插槽内部定义提示文字slot=tip，上传文件slot=file内容等，通过作用域插槽解构出上传的内容，从而获取路径等参数进行预览或者删除等操作slot-scope={file}，在上传文件插槽中可以定义加载状态v-loading防止文件未加载完成就点击预览从而导致预览失败的问题;
  • （图片上传）图片预览可以使用el-image进行大图预览，但是要注意previewList(图片预览数组)有多个图片时图片位置以及删除时的顺序问题；
  • 前端使用axios的post请求进行文件上传，最后会返回一个服务器存放的文件路径，我们把返回的数据拼装成 el-upload组件绑定的fileList需要的格式，也就是{name:string, url:string}的对象数组；
  • 文件预览方面，excel与pdf使用window.open(url, '_blank')方式就能在实现预览下载功能;但是excel文件下载需要指定下载文件名称的话，需要 XMLHttpRequest 请求，通过get请求方式执行下载操作，在 XMLHttpRequest 实例的onload配置文件名称等属性值；

4.1.2 注意点以及优化项

  • 首先，这边定义 auto-upload属性设置为false，那么文件就不是自动上传的方式了，所以官方文档中的一些钩子函数例如before-upload,on-success等都会失效，这一点源码中也能看出，这些钩子都是在 auto-upload=true的情况下才会执行的逻辑；我们需要把原先的校验逻辑，上传前数据转换等待放在on-change函数当中。
  • 数据回显方面el-upload组件内部支持由{name: string, url: string}构成的对象数组，这一点需要状态层的人员注意，在定义数据结构时尽量定义成这种，否则的话ui组件双向绑定时需要做兼容处理；兼容方式为父组件传入一个对象结构fileMap={name: fileName, url: filePath},其中fileName与filePath为业务层给的双向绑定的字段;在数据回显的逻辑中给双向绑定的数组里面的每一项都加入fileName与filePath字段
  • 其次，预览时默认给的el-dialog内部嵌套img标签展示图片的方式，这种方式在上传图片过大过小时展示会出现一些布局上的问题，比如不能居中或者说需要依靠滚动栏才能看清整个图片，用户体验比较差。针对这个，我们可以换成el-image大图预览，这种方式可以展示出整张图片，自带的功能中可以支持放大、缩小并且在上传多张图片时能够支持上下切换。
  • 但是使用el-iamge多图预览中，我们需要考虑预览图片的数组，因为这个数组时静态的数据源，每次点击时只会展示数组第一项，解决方式就是在点击预览按钮时，将选中的当前项从预览数组中置顶；当然，如果需求是只支持单个图片的情况就不需要考虑了。
  • 图片上传过程中，有可能因为图片过大上传比较慢的原因，导致图片还处于上传阶段但是用户点击了预览按钮，这样就会导致图片展示失败，让人误以为是文件上传失败其实是文件还处于上传当中；解决方式就是在 <div slot=file>标签图展示图片的<el-image>加上加载状态v-loading,保证上传成功才开放预览删除等功能
  • 图片上传过程中，我们常常需要对图片格式，尺寸进行大小或是文件后缀的拦截，当文件过大或是不合格的文件被上传，虽然代码逻辑控制了不走后端上传请求，但是样式上图片已经占据了上传位置，然后用户也无法删除或是其他操作；解决方式就是在上传了不合格的图片文件时，将该项文件从绑定的 fileList数组中移除；
  • 对于只能上传单个图片的需求，我们可以通过自定义类名隐藏上传的输入框，因为官网的limit属性只会在逻辑上控制文件的不可上传，但是对于用户来说，直接隐藏上传框会更加直接；解决方式就是给<el-upload>标签挂一个类名，根据limit设置的数量与绑定fileList的数组（也就是传给后端的参数）进行比较，达到限制就display:none隐藏
  • 以上就是，基于el-upload组件进行图片上传的一些要点以及用户体验优化了，在体验优化这块，有时候需要和pm确认一下，并且做出这些优化往往需要细致的测试，否则容易产生一些问题。

上传图片事件uploadImgChange

  • ElUploadInternalFileDetail、ElUploadInternalFileDetail自定义文件以及文件列表结构用于类型推断
  • 添加加载状态，进行文件格式，大小等校验逻辑
  • 使用FileReader构造函数实例化出reader,使用实例readAsDataURL方法接收一个Blob类型的入参并且在函数执行完成拿到后端需要的base64文件数据，通过split等方法把文件base64从返回结果中分割出来，因为后端的入参只需要这部分；
uploadImgChange(file: ElUploadInternalFileDetail, fileList: Array<ElUploadInternalFileDetail>) {
      this.$set(this.loading, file?.uid, true)
      const imgReg = /\.jpg|\.bmp|\.png$/i
      if (imgReg.test(file?.name)) {
        const reader = new FileReader()
        reader.readAsDataURL((file.raw) as any)
        reader.onloadend = () => {
          const dataURL = reader.result as string
          this.uploadFileWithDataUrl(dataURL, fileList, file)
        }
      } else {
        // 上传失败的文件在上传的列表数组中移除
        const idx = fileList.findIndex((item: ElUploadInternalFileDetail) => {
          return item.url === file.url
        })
        fileList.splice(idx, 1)
        this.$message.warning('请上传指定后缀的图片！')
        return
      }
    }
  • uploadFileWithDataUrl方法内部就是执行上传文件至后端服务的逻辑，使用到的是axios.post请求
  • 使用方式形如：
const params = {} //这是后端入参
const baseUrl = 'xxxx' //这是后端接口
try {
     axios.post(baseUrl, param).then(data => {
         // 这里上传成功后返回文件路径以及文件名称,将返回的数据推入预览数组以及双向绑定的数组
         file[this.fieldMap.url] = data.data.data
         file[this.fieldMap.name] = param.fileName
         this.imgList.push(file)
         this.previewList.push(file[this.fieldMap.url])
         this.$emit('input', this.imgList)
         this.$set(this.loading, file.uid, false)
     })
 } catch (error) {
     this.$message('图片转换失败')
 }

4.2 上传pdf文件

  • pdf的文件上传与上文图片上传大体类似，例如数组转换，接口请求都是一致的，唯一区别就是不需要考虑预览的情况，在没有其他要求比如指定下载文件名称等需求时，预览只需要window.open(url, '_blank')

4.3 上传excel文件

  • excel文件的上传则与pdf上传类似，但是这里pm指定了下载文件的名称，所以原本简单粗暴的window.open()的文件下载方式行不通了，这里使用XMLHttpRequest发送get请求，使用方式形如：
downloadFile(file: any) {
    //实例化出一个 xhr
    const xhr = new XMLHttpRequest()
    // 初始化一个get请求 
    xhr.open('GET', file.url, true)
    // 指定返回类型
    xhr.responseType = 'blob'
    xhr.onload = () => {
        const element = document.createElement('a')
        element.style.display = 'none'
        //创建下载链接
        const url = window.URL.createObjectURL(xhr.response)
        element.href = url
        // 设置下载的文件名
        element.download = '自定义文件名.xls'
        element.click()
        document.body.removeChild(element)
        // 释放内存
        window.URL.revokeObjectURL(url)
    }
    xhr.send()
}

4.4 上传视频文件（video）

1.上传

  • 上传视频逻辑与上面的差异只存在于，预览时会使用到原生的 video标签（在没有指定需求的情况下）；
  • 但是,这里需要将视频文件上传至外网环境，这就导致原本的接口请求地址以及传参方式有所区别；主要体现在请求体需要是 FormData格式的对象作为请求载体，写法格式；
 const param = new FormData()
 param.append('file', file.raw)
 param.append('context', url)

  • 后端需要的是file数据为Blob类型，这里使用el-upload上传之后的file对象中的raw对象直接作为入参就可；

  • 此外，入参还需要一个上下文参数，这个context在前端配置中获取到的是base64格式，通过上面的方法解码之后才能作为入参 ，至此请求参数就完整了；剩下的就是执行上传请求,使用起来形如： axios.post(url, params)

  • 经过上面的步骤，请求成功之后返回视频存放的资源地址，接下来就是预览视频相关的问题了。。。

2.预览

  • 预览方式是在上传组件中，点击预览按钮打开一个el-dialog弹窗，咱们把video标签内嵌在弹窗当中，在打开之后将video标签内的src属性赋值，也就是刚刚我们上传请求后返回的资源地址；
  • 关于video标签上面一开始也有提到，内置有很多属性，比如自动播放，展示控制面板。。。
  • 这里有一个问题，一开始的处理方式是组件内部定义了一个播放方法，父组件通过ref拿到el-dialog实例从而触发定义好的playVideo方法;但是控制台却提示找不到相关方法 play,pause;在playVideo方法中输出videoPlayer实例为undefined
  • 最终发现，由于video是内嵌在了el-dialog中，而el-dialog中存放的标签内容在源代码中是通过v-if进行显隐控制的，所以在 弹窗刚展示isShow=true时就去获取video实例的话就会发生上面的情况，解决方式就是使用this.$nextTick延迟回调，保证在video实例存在后才执行实例方法；
//父组件
<template>
    <div>
        <el-upload></el-upload>
        // 播放子组件
        <preview-video ref='previewVideoRef'></preview-video>
    </div>
</template>

//播放子组件
<template>
    <div>
        <el-dialog
              width="1000px"
              class="m-dialog m-video"
              :visible.sync="isShow"
              :close-on-click-modal="false"
            >
           <video ref="videoPlayer" class="video-cont" :src="videoSrc" 
                  autoplay controls  controlslist="nodownload"></video>
        </el-dialog>
   </div>
</template>
<script lang="ts">
    import { Component, Ref, Vue } from 'vue-property-decorator
    @Component
    export default class extends Vue {
        @Ref('videoPlayer') videoPlayer: any
        videoSrc = '' 
    isShow = true
    playVideo(src: string) {
        this.isShow = true
        // 修改后
        this.$nextTick(() => {
            this.videoPlayer.play()            
        })
    }
    }
</script>
