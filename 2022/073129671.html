Js之this

JavaScript 上下文
发布日期:   2022-07-31
文章字数:   1.9k
阅读时长:   8 分

1. 基本认知

  • 大多数情况下，函数的调用方式决定了this的值
  • this不能在执行期间被赋值，每次函数被调用时this的值可能不同
  • es5可以通过bind方法设置函数this
  • es6引入箭头函数，箭头函数不提供自身的this(this的值将保持为闭合词法上下文的值)

2. this 语法

  • this是当前执行上下文（global,function, eval）的一个属性，非严格模式下总是指向一个对象，严格模式下可以是任意值
const test = {
    prop: 42,
    func: function () {
        return this.prop
    },
}
// this 指向的是 test对象
console.log(test.func())
// expected output: 42

2.1 全局上下文

  • 全局上下文，不论是否在严格模式下，在全局执行环境中（函数体外部），this都指向全局对象
  • 可以使用 globalThis 获取全局对象，无论你的代码是否在当前上下文运行

2.1.1 浏览器中的全局对象

  • 浏览器中，window对象就是全局对象
console.log(this === window) // true
a = 88
console.log(window.a) // 88
console.log(globalThis, '获取全局对象')

2.2 函数上下文

  • 在函数内部，this的值取决于函数被调用的方式

2.2.1非严格模式下：

function f1() {
    return this
}
//在浏览器中：
console.log('brower', f1() === window) //在浏览器中，全局对象是 window

//在 Node 中：
console.log('Node', f1() === globalThis) // globalThis有着定时器setTimeout等内置方法

2.2.2 严格模式下

  • 如果进入执行环境时没有设置 this 的值，this 会保持为 undefined
  • 想要把 this的值从一个环境传到另一个，需要call,apply方法
  • 以下栗子因为 f2函数 是被直接调用的，而不是作为对象的属性或方法调用的（如 window.f2()）
function f2() {
    'use strict' // 这里是严格模式
    return this
}
console.log('严格模式：', f2() === undefined) // true

2.3 类上下文

2.3.1 基本概念

  • this在类中与函数中类似，类的本质也是函数，但是也还有一定区别
  • 在类的构造函数中，this是一个常规对象，类中所有非静态方法都会被添加到this的原型中
  • 静态方法不是this的属性，他们只是类自身的属性

代码栗子：

class Example {
    constructor() {
        const proto = Object.getPrototypeOf(this)
        console.log('属性名：', Object.getOwnPropertyNames(proto))
    }
    first() {}
    second() {}
    // 静态方法只是类自身的属性
    static third() {}
}

const obj = new Example() // ['constructor', 'first', 'second']
console.log('实例化：', obj)

2.3.2 派生类

  • 派生类不像基类的构造函数，派生类的构造函数没有初始的this绑定；
  • 在构造函数中调用super()会生成一个this绑定,等价于在派生类中实例化一个基类实例;形如：this = new Base()
  • super()之前引用this会抛出错误
  • 派生类不能在调用super之前返回，除非构造函数返回的是一个对象或者没有构造函数

代码栗子：

class Base {}
class Good extends Base {}
class AlsoGood extends Base {
    constructor() {
        return { a: 5 }
    }
}
//作为 Base的派生类，没有调用 super并且存在构造函数的情况下没有返回对象；实例化对象时报错
class Bad extends Base {
    constructor() {}
}
console.log('------start------')
console.log(new Good())
console.log(new AlsoGood())
console.log(new Bad()) // 报错 Uncaught ReferenceError
console.log('----------end--------')

3. this和对象转换

  • 非严格模式下使用 call 和 apply 时，如果用作 this 的值不是对象，则会被尝试转换为对象。
  • null 和 undefined 被转换为全局对象
  • 原始值如 7 或 ‘foo’ 会使用相应构造函数转换为对象。因此 7 会被转换为 new Number(7) 生成的对象，字符串 ‘foo’ 会转换为 new String('foo') 生成的对象

3.1 call

function bar() {
    console.log(Object.prototype.toString.call(this))
}

bar.call(7) // [object Number]
bar.call('foo') // [object String]
bar.call(undefined) // [object global]

3.1 bind

  • 调用f.bind(someObject)会创建一个与 f 具有相同函数体和作用域的函数
  • 但是在这个新函数中，this将永久地被绑定到了bind的第一个参数，无论这个函数是如何被调用的。
 function f() {
     return this.a
 }

var g = f.bind({ a: 'azerty' })
console.log(g()) // azerty

var h = g.bind({ a: 'yoo' }) // bind 只生效一次！
console.log(h()) // azerty

4. 箭头函数

4.1 箭头函数中的this

  • 在箭头函数中，this与封闭词法环境的this保持一致
  • 在全局代码中，它将被设置为全局对象：
var globalObject = this
var kkk = () => this
console.log('箭头：', kkk() === globalObject) // true

 // 1. 如果将this传递给call、bind、或者apply来调用箭头函数，它将被忽略。
// 2. 不过你仍然可以为调用添加参数，不过第一个参数（thisArg）应该设置为null。
var obj2 = { kkk2: kkk }
console.log(
    'k...',
    obj2.kkk2() === globalThis
) //true

// 使用call来设置this
console.log(
    'call...',
    kkk.call(obj2) === globalThis
)
// 使用bind来设置this
ppp = kkk.bind(obj2)
console.log(
    'bind...',
    ppp() === globalThis
)

4.2 普通函数使用箭头函数

  • 箭头函数this，永久绑定到它外层函数的this
var temp = {
    bar: function () {
        var x = () => this
        return x
    },
}
  • 调用temp中bar函数，将它的this绑定到temp
  • 将返回函数的引用赋值给fn

栗子形如：

var fn = temp.bar()
console.log('temp...', fn() === temp)
  • 但是如果上面的 fn只是引用了temp的方法，并没有调用，那么this指向的就是全局对象
  • 这里调用之后，fn1的this继承自bar

栗子形如：

var fn1 = temp.bar
// 输出fn1，得到是bar函数， fn1()得到的是 ()=>this, fn1()()得到的是全局对象
console.log('temp。。。', fn1()())

5.(原型链、构造函数、类…)中的this

5.1 原型链中的this

  • 代码栗子
 var o = {
     f: function () {
         return this.a + this.b
     },
 }
 // 用于创建一个新对象，使用现有对象作为新创建对象的原型
 var p = Object.create(o)
 p.a = 1
 p.b = 4
// 对象 p 没有属于它自己的 f 属性，它的 f 属性继承自它的原型
console.log('这是p。。。。', p.f()) // 5

5.2 getter与setter中的this

  • 用作 getter 或 setter 的函数都会把 this 绑定到设置或获取属性的对象
function sum() {
    return this.a + this.b + this.c
}
var o = {
    a: 1,
    b: 2,
    c: 3,
    get average() {
        return (
            (this.a + this.b + this.c) /
            3
        )
    },
}

Object.defineProperty(o, 'sum', {
    get: sum,
    enumerable: true,
    configurable: true,
})

console.log(o.average, o.sum) // logs 2, 6

5.3 构造函数中的this

  • 虽然构造函数返回的默认值是 this 所指的那个对象，
  • 但它仍可以手动返回（也就是构造函数内 return出一个对象）其他的对象（如果返回值不是一个对象，则返回 this 对象）
  • 栗子：
function MyConstructor(){
    // 函数实体写在这里
    // 根据需要在 this 上创建属性，然后赋值给它们，比如：
    this.fum = "nom";
    // 等等...
    // 如果函数具有返回对象的 return 语句，return返回的对象将是 new 表达式的结果。
    // 否则，表达式的结果是当前绑定到 this 的对象。
}
  • 代码栗子
function C() {
    this.a = 37
}
var o = new C()
console.log(o.a) // logs 37
function C2() {
    this.a = 37
    // return了对象之后，this.a=37相当于被废弃了，但是不会对外界造成影响
    return { a: 38 }
}
o = new C2()
console.log(o.a) // logs 38

5.4 类中的this

  • 类和其他普通函数一样，方法中的 this 值取决于它们如何被调用。
  • 也可以通过bind方法让类中的 this 值总是指向这个类实例

代码形如：

 class Car {
     constructor() {
         // 使用bind 绑定sayBye方法
         this.sayBye = this.sayBye.bind(this)
     }
     sayHi() {
         console.log(
             `Hello from ${this.name}`
         )
     }
     sayBye() {
         console.log(
             `Bye from ${this.name}`
         )
     }
     get name() {
         return 'Ferrari'
     }
 }

class Bird {
    get name() {
        return 'Tweety'
    }
}

const car = new Car()
const bird = new Bird()

car.sayHi() // Hello from Ferrari
bird.sayHi = car.sayHi
bird.sayHi() // Hello from Tweety

// sayBye方法在 Car类中通过bind绑定了this指向
bird.sayBye = car.sayBye
bird.sayBye() // 输出了 Bye from Ferrari 而不是 Tweety
