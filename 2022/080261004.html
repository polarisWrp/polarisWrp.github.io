Js之作用域链

JavaScript 作用域
发布日期:   2022-08-02
文章字数:   2.4k
阅读时长:   9 分

一、静态作用域

1.1 代码栗子

<script>
    var value = 1
    function foo() {
        console.log(value)
    }
    function bar() {
        var value = 2
        foo()
    }
    bar() // 1
</script>
  • Js采用静态作用域（词法作用域）执行foo函数，先从foo函数内部查找发现没有value变量；接着根据书写的位置，查找上面一层的代码，找到了外层的value=1,所以bar函数执行输出 1

1.2 代码栗子2

  • JavaScript 函数的执行用到了作用域链，这个作用域链是在函数定义的时候创建的；
  • 嵌套的函数f定义在这个作用域链里，其中的变量scope一定是局部变量；
  • 不管何时何地执行函数 f，这种绑定在执行 f 时依然有效。
<script>
    var scope = 'global scope'
    function checkscope() {
        var scope = 'local scope'
        function f() {
            return scope
        }
        return f()
    }
    const res = checkscope()
    console.log(res) // local scope
</script>

1.3 代码栗子3

  • Js函数执行时一块一块执行
  • foo变量提升
<script>
    var foo = function () {
        console.log('foo1')
    }
    foo() // foo1

    var foo = function () {
        console.log('foo2')
    }
    foo() // foo2
</script>
/* --------------以上等价于-------------- */
<script>
    var foo 
    foo = function() {
        console.log('foo1');
    }
    foo()
    foo = function() {
        console.log('foo2');
    }
    foo()
</script>
  • 函数提升
<script>
    function foo() {
        console.log('foo111')
    }
    foo() // foo2

    function foo() {
        console.log('foo222')
    }
    foo() // foo2
</script>
/* ----------等价于--------- */
<script>
    function foo() {
        console.log('foo111')
    }
    function foo() {
        console.log('foo222')
    }
    foo() // foo222
    foo() // foo222
</script>

二、执行上下文

1. 定义：

  • js 可执行代码分为 全局代码，函数代码，eval代码 ；
  • 当执行一个函数时，就会创建执行上下文, Js引擎通过执行上下文栈来管理执行上下文 ；
  • 当Js代码每次执行一段可执行代码时，都会创建对应的执行上下文；
  • 每个执行上下文都有三个重要属性 1.变量对象（variable object VO）2.作用域链（scope chain）3.this

2. 代码栗子

2.1 demo1

  • 执行流程:
    1. 初始化向执行上下文栈压入全局执行上下文 globalContext
    2. 依次向栈中压入 fun1 fun2 fun3; 依次执行后弹出fun3 fun2 fun1
<script>
    var ECSstack = [] //执行上下文栈
    function fun3() {
        console.log('fun3')
    }
    function fun2() {
        fun3()
    }
    function fun1() {
        fun2()
    }
    fun1()
</script>

2.2 demo2

  • 向全局上下文栈压入globalContext, checkscope, f
  • 执行之后弹出f, checkscope, globalContext
<script>
    var scope = 'global scope'
    function checkscope() {
        var scope = 'local scope'
        function f() {
            return scope
        }
        return f()
    }
    checkscope()
</script>

2.3 demo3

  • 与demo2不同的是，函数checkscope返回的是变量f
  • 执行流程变为，执行上下文栈压入globalContext,checkscope,执行后弹出checkscope,压入函数f，执行函数f后弹出f,globalContext
<script>
    var scope = 'global scope'
    function checkscope() {
        var scope = 'local scope'
        function f() {
            return scope
        }
        return f
    }
    checkscope()() // local scope
</script>

三、执行上下文之变量对象

  • 变量对象是与执行上下文相关的数据作用域，存储了在上下文中定义的 变量和函数声明;
  • 不同执行上下文的变量对象不相同；

1. 全局上下文的变量对象

  • 全局上下文中的变量对象就是全局对象！在客户端中全局对象即为window对象。

  • 全局上下文（全局对象）

    1. 是预定义的对象，可以访问所有其他预定义的对象、函数和属性；
    2. 在Js顶层代码中，可以通过this引用全局对象，全局对象是作用域链的头，所以所有非限定的变量和函数名都会作为该对象的属性来查询；
    3. 例如： 当js代码引用parseInt()函数时，它引用的是全局对象的parseInt属性,作为作用域链的头，所有Js顶层声明的变量都会成为全局对象的属性。

  • 全局对象

    1. 在客户端中，通过this引用全局对象即window对象；
    2. 全局对象是由Object构造函数实例化的一个对象，this instanceof Object返回true；
    3. 预定义了很多的属性和方法，使用形如：Math.random(),this.Math.random()；
    4. 作为全局变量的宿主，var a = 1,this.a；
    5. 在客户端中，全局对象有window属性指向本身，栗子形如： var a=1 console.log(window.a);this.window.b=2 console.log(this.b)。

2. 函数上下文的变量对象

2.2.1 定义：

  • 在函数上下文中，用活动对象（activation object, AO）表示变量对象;
  • 即活动对象 === 变量对象，但是变量对象是规范上的或者说是引擎实现的，不可在js环境中访问；
  • 当进入一个执行上下文中，执行上下文的变量对象才被激活，所以叫做活动对象，并且也只有活动对象的各种属性可以被访问使用。

2.2.2 执行过程

  • 执行过程分为： 代码分析与处理
  • a.进入执行上下文 b.代码执行
a. 进入执行上下文
  • 此时的代码还未执行，变量对象包括：
    1. 函数所有形参（函数上下文）
      • 由名称和对应值组成的一个变量对象的属性被创建
      • 没有实参，属性值为undefined
    2. 函数声明
      • 由名称和对应值组成的一个变量对象的属性被创建
      • 如果变量对象已经存在同名的属性，则完全替换这个存在的属性
    3. 变量声明
      • 由变量和对应值组成的一个变量对象的属性被创建
      • 如果变量名称和已经声明的形参或者函数名相同，则变量声明不会干扰已经存在的这类属性
代码栗子1
<script>
    function foo(a) {
        var b = 2
        function c() {}
        var d = function () {}
        b = 3
    }
    foo(1)
</script>
<!-- 进入执行上下文后 -->
AO = {
    arguments: {
        0: 1,
        length: 1
    },
    a: 1,
    b: undefined,
    c: reference to function c(){},
    d: undefined,
}
<!-- 执行代码阶段会给变量再次赋值 --> 
AO = {
    rguments: {
        0: 1,
        length: 1
    },
    a: 1,
    b: 3,
    c: reference to function c(){},
    d: reference to FunctionExpression "d"
}

2.2.3 总结

  • 全局上下文中的变量对象初始化是全局对象；
  • 函数上下文的变量对象初始化只包括 Arguments对象；
  • 在进入执行上下文时会给变量对象添加形参，函数声明以及初始的属性值；
  • 代码执行阶段，会再次修改变量对象的属性值。
总结栗子：
栗子1
<script>
    function foo() {
        console.log(a)
        a = 1
    }
    foo() 
    // 第一步a中变量没有声明，所以控制台打印 a is not defined
    // 首先在函数内部查找，内部没有就去全局查找，依然没有则报错
    // 1. 进入执行上下文时，变量对象
    /* 
        AO = {
          arguments: {
            length: 0
          }
        } 
      */
</script>
栗子2
  • 函数foo因为变量a还未定义就执行输出代码，所以控制台a is not defined
  • 执行函数bar时，由于foo函数中执行a=1，全局对象中存在a变量值为1，所以函数bar中执行输出语句时在全局对象中查找到了a变量，因此输出1
<script>
    function foo() {
        console.log(a)
        a = 1
    }
    foo() // a is not defined
    function bar() {
        a = 1
        console.log(a)
    }
    bar() // 1
</script>
栗子3
  • 函数kkk声明提前，然后才是变量kkk提升；当变量名称跟已经声明的形参或函数相同，变量声明不会干扰已经存在的这类属性，因此首行输出函数kkk
<script>
    console.log(kkk) //打印函数
    function kkk() {
        console.log('kkk')
    }
    var kkk = 1
</script>

四、执行上下文之作用域链

1. 定义

  • 当查找变量时，会先从当前上下文中的变量对象中查找；找不到的情况，会从父级变量对象中查找；还找不到的情况下，则从全局对象中查找；
  • 这样由多个执行上下文的变量对象构成的链表叫作用域链

2. 函数创建

  • 函数的作用域在函数定义时就决定了；
  • 函数内部有一个 [[scope]]属性，当函数创建时，会保存所有父变量对象到其中；
    1. 理解为[[scope]]是所有父变量对象的层级链
    2. 但是： [[scope]]不是表示完整的作用域链

2.1 代码栗子

<script>
    function foo() {
        function bar() {
            // ...
        }
    }

    // 函数创建时，各自的[[scope]]
    /* 
          foo.[[scope]] = [
            globalContext.VO
          ]

          bar.[[sccope]] = [
            fooContext.AO
            globalContext.VO
          ]
      */
</script>

3. 函数激活

  • 进入函数上下文时，创建 VO/AO,并将活动对象AO添加到作用域链的顶端；
  • 此时的执行上下文作用域链 Scope = [AO].concat([[Scope]])；

4.完整执行流程栗子

<script>
    var scope = 'global scope'
    function checkscope() {
        var scope2 = 'local scope'
        return scope2
    }
    checkscope()
</script>
4.1 函数创建
// 1. checkscope 函数创建，保存父级变量对象(这里是全局对象)层级链
checkscope.[[scope]] = [
    globalContext.VO
]
4.2 函数被调用
// 2. 创建函数执行上下文，将函数上下文(这里是checkscopeContext)压入执行上下文栈
ECSstack = [
    checkscopeContext,
    globalContext
]
4.3 函数执行前的准备工作
  1. 复制函数 [[scope]]属性创建作用域链
  2. 用 arguments创建活动对象，初始化活动对象，加入形参、函数声明以及变量声明
  3. 将活动对象压入 checkscope 作用域链顶端
1. 复制函数 [[scope]]属性创建作用域链
checkscopeContext = {
    Scope: checkscope.[[scope]]
}
2. 用 arguments创建活动对象，初始化活动对象，加入形参、函数声明以及变量声明
checkscopeContext = {
    AO: {
        arguments: {
            length: 0
        },
        scope2: undefined
    },
    Scope: checkscope.[[scope]]
} 
3. 将活动对象压入 checkscope 作用域链顶端
checkscopeContext = {
    AO: {
        arguments: {
            length: 0
        },
        scope2: undefined
    },
    Scope: [AO, checkscope.[[scope]]]
}
4.4 准备结束，执行函数,修改AO属性值
checkscopeContext = {
    AO: {
        arguments: {
          length: 0
        },
        scope2: 'local scope'
    },
    Scope: [AO, [[Scope]]]
}
4.5 函数执行完毕，函数上下文从执行上下文栈中弹出
ECSstack = [
    globalContext
]
